-Revisar constructores que tienen id en el modelo, la id autoincrementable, 
te comentarie uno para que veas.

-El archivo bd-restaurant.mwb es el modelo, tenes que aplicar 
Forward Engineer desde el menu Database para poder levantar
el modelo en Local Instance donde estan todas las SCHEMAS como por
ejemplo la base sakila que viene por defecto con mysql. Vas a encontrar 
algunos errores en este proceso como por ejemplo la tabla mesa
con dos idMesa. Una vez que te aparece mydb en SCHEMAS(desde Local Instance)
recien ahí vas a poder cargarla con datos de prueba, cuando termines vas
a tener que seleccionar en MANAGEMENT la opcion Data Export
para generar el archivo .sql con la base con los datos que cargaste
y así poder trabajarla sin tener que insertar datos cada vez que importamos
la base. Metele todas las fichas a tener la base con datos de prueba antes 
de seguir con los mapeos porque nos sirven para avanzar con las vistas.

-Para agilizar el proceso de generacion de datos de prueba te dejo esta pagina
donde podes generar un archivo cvs con 100 row de manera automatica.

http://www.generatedata.com/

-Ese archivo lo cargas como los copy del postgresql con el siguiente comando.

	LOAD DATA LOCAL INFILE 'C:\archivo.csv'
	INTO TABLE nombre_tabla
	FIELDS TERMINATED BY ';'
	LINES TERMINATED BY '\n'
	(campo1, campo2, campo3);

Acordate poner los archivos .csv en c:\ y guardarlo como archivo delimitado por ;
